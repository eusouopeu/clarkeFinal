Integração Contínua

É uma prática de automatizar a integração de alterações de código de vários contribuidores em um único projeto de software, permitindo que os desenvolvedores mesclem com frequência as alterações de código em um repositório central onde builds e testes são executados

Algumas ferramentas de Integração são:

  • Jenkins
  • Azure DevOps
  • CodeShip
  • PagerDuty

Dentre as vantagens de seu uso, estão:

  • Melhorar a produtividade do desenvolvedor
  • Encontrar e investigar bugs mais rapidamente
  • Distribuir atualizações mais rapidamente

Arquitetura Modular

Arquitetura modular é uma abordagem de design de sistemas onde um sistema é dividido em módulos menores, independentes e reutilizáveis, que podem ser desenvolvidos, testados e mantidos separadamente.

Principais Características: O sistema é dividido em módulos, cada um com uma funcionalidade específica, onde os módulos podem ser desenvolvidos e atualizados de forma independente, podem ser reutilizados em diferentes sistemas ou projetos. Problemas podem ser identificados e corrigidos em módulos específicos, sem afetar todo o sistema e permite a substituição ou atualização de módulos sem grandes impactos no sistema geral.

Vantagens:

  • Escalabilidade
  • Manutenibilidade
  • Colaboração
  • Eficiência no Desenvolvimento

Testes Unitários

Verificam a funcionalidade de unidades individuais de código, como funções, métodos ou classes, com o objetivo dearantir que cada unidade funcione corretamente isoladamente.

Cada teste é executado de forma independente, sem dependências externas. Além de serem rápidos de executar, permitindo a detecção precoce de erros, facilitam a automação e são geralmente executados frequentemente durante o desenvolvimento.

Uma ferramenta para teste unitário que a Clarke utiliza é o Python

5
Testes de Integração

Avaliam a interação entre diferentes módulos ou componentes de um sistema para garantir que os módulos funcionem corretamente quando combinados. São focados nas interfaces e nas interações entre os módulos e geralmente mais complexos e demorados que os testes unitários. Verificam a integração de componentes maiores e a comunicação entre eles

Ferramentas Comuns:

  • TestNG (Java)
  • MSTest (.NET)
  • Selenium (para integração de frontend)

Banco de Dados

Banco de Dados é o sistema de armazenamento de informações que permite a organização, gerenciamento e recuperação de dados de uma empresa. Para acessá-lo, geralmente utiliza-se linguagens de programação especificas, chamadas linguagens de acesso ao Banco de Dados. Essas linguagens, por sua vez, podem ser classificadas como: Relacionais e Não-Relacionais

A modelagem, a manutenção, a segurança, o acesso e a manipulação dos dados dos bancos de dados geralmente faz parte das responsabilidades de um DBA (Administrador de Banco de Dados) ou de um desenvolvedor de back-end.

As linguagens de acesso ao Banco de Dados usadas pela Clarke são:

  • PostgreSQL (relacional)
  • MongoDB (não relacional)

Banco de Dados Relacional: PostgreSQL

Sistema de gerenciamento de banco de dados relacional avançado e open-source. Ele organiza os dados em tabelas estruturadas, onde cada tabela é composta por linhas e colunas.

Características Principais:

  • Linguagem SQL (Structured Query Language)
  • Esquema Rígido
  • Chaves Primárias e Estrangeiras
  • Transações ACID

Hospedagem

Hospedagem de software refere-se ao serviço de disponibilização de aplicações e sites na internet. Empresas ou provedores de hospedagem oferecem infraestrutura e serviços para armazenar, executar e manter esses softwares acessíveis online.

O tipo de hospedagem utiliado pela Clarke é em nuvem, mais especificamente o Google Cloud Platform.

Hospedagem em nuvem é um serviço que utiliza múltiplos servidores interconectados para hospedar aplicações e sites, oferecendo maior flexibilidade, escalabilidade e desempenho comparado aos métodos tradicionais de hospedagem. É um tipo de hospedagem escalável, automática, de alta disponibilidade e segurança e de ótimo custo benefício

Aplicativo Mobile

  • Multiplataforma: disponível tanto na Apple Store (dispositivos iOS) quanto na Play Store (dispositivos Android)
  • Linguagens: Swift (iOS), Java (Android), React Native / Flutter (multiplataformas)

Políticas de Backup &

A política de backup refere-se às práticas e procedimentos estabelecidos para garantir que os dados importantes sejam protegidos e possam ser recuperados em caso de perda ou corrupção.

A Clarke foca em duas principais políticas de backup:

  • Backup Diário do Banco de Produção:
    Significa que cópias de segurança (backups) dos bancos de dados usados em um ambiente de produção, onde as operações reais e transações de usuários ocorrem são feitas todos os dias.
  • Versionamento do GitHub:
    Significa que o código-fonte é atualizado e versionado no GitHub diariamente. Isso geralmente envolve commits regulares e pushs para o repositório no GitHub.
  • Diferencial: Cópia dos dados alterados desde o último backup completo; balanceia eficiência e rapidez na restauração.

A política de backup existe para garantir prontidão para lidar com perdas de dados, evitar prejuízos financeiros e perda de credibilidade e permite restauração rápida e precisa em caso de incidentes. Isso porque a LGPD exige o uso de servidor seguro e criptografado, que favorece a exclusão de dados dados conforme solicitado e detecta aplicações não autorizadas.

Componentes de uma Política de Backup:

  • Escopo e Objetivos: Dados e sistemas incluídos, RTO e RPO.
  • Frequência de Backup: Baseada na criticidade dos dados.
  • Tipos de Backup: Completo, incremental, diferencial.
  • Métodos de Armazenamento: Locais, remotos ou em nuvem.
  • Procedimentos de Recuperação: Definição clara de procedimentos e responsabilidades.
  • Testes e Auditorias: Frequência de testes e auditorias.
  • Segurança:: Proteção dos dados de backup.
